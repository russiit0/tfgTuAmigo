I

Análisis de Requisitos

ℹ️

Tu Amigo es una aplicación de escritorio multiplataforma cuyo propósito es ofrecer un espacio seguro y confidencial donde estudiantes que sufren acoso escolar puedan expresarse libremente y recibir apoyo emocional inmediato a través de un asistente de inteligencia artificial empático.

Requisitos Funcionales

ID Descripción Prioridad
RF-01El sistema debe ofrecer un chat de soporte emocional impulsado por IA accesible desde la pantalla de bienvenida.Alta
RF-02El sistema debe mostrar recursos de emergencia permanentemente disponibles: número 112 y línea ANAR 900 20 20 10.Alta
RF-03La inteligencia artificial debe generar una respuesta empática en menos de 5 segundos bajo condiciones normales de red.Alta
RF-04El sistema debe permitir visualizar una simulación educativa de conversación entre dos agentes IA (víctima y asistente).Media
RF-05La simulación debe mostrar métricas emocionales en tiempo real: tristeza, ansiedad, alivio y esperanza (escala 0–100).Media
RF-06El sistema debe extraer automáticamente un perfil del estudiante durante la simulación: nombre, edad, nivel de riesgo y acción recomendada.Media
RF-07El sistema debe permitir guardar la sesión de simulación completa como archivo JSON descargable.Baja
RF-08La aplicación debe estar disponible y funcionar correctamente en Windows, macOS y Linux.Alta

II

Diseño Físico y Lógico

Arquitectura del Sistema

La aplicación sigue un modelo de tres capas separadas físicamente: la capa de presentación (React en el proceso renderer de Electron), la capa de lógica/proceso (Node.js en el proceso principal de Electron) y el servicio externo de inteligencia artificial (Google Gemini). La comunicación entre las dos primeras capas se realiza de forma segura mediante IPC (Inter-Process Communication).

🖥️
Capa de Presentación — React Renderer (Chromium)

React 19 + TypeScript + TailwindCSS · WelcomeScreen · ChatScreen · ResourcesScreen · SimulationScreen

↕ IPC segura · window.electronAPI.chat() · contextBridge
⚙️
Capa de Proceso Principal — Electron Main (Node.js)

main.ts · ipcMain.handle('chat') · Normalización de historial · Gestión de ventana

↕ HTTPS / REST · SDK @google/generative-ai
🤖
Servicio Externo de IA — Google Gemini 2.0 Flash

Generación de respuestas empáticas · Análisis de perfil · Métricas emocionales · Simulación multi-agente

CU-01

Iniciar conversación de apoyo emocional

Descripción El estudiante inicia una conversación con el asistente IA para recibir apoyo emocional ante una situación de acoso escolar. El sistema responde de forma empática, sin juzgar, adaptando el tono al estado emocional percibido.
Actores Estudiante (usuario principal de la app)
Precondiciones La aplicación está abierta y existe conexión a internet para acceder a la API de Gemini.
Flujo normal 1. El estudiante pulsa "Hablar con alguien" en la pantalla de bienvenida.
2. Se carga la pantalla de chat con un mensaje inicial de la IA: "Hola. Siento que estés pasando por un momento difícil. ¿Qué es lo que más te preocupa hoy?"
3. El estudiante escribe su mensaje y pulsa enviar.
4. La app envía el historial de mensajes al proceso principal de Electron mediante IPC.
5. El proceso principal llama a la API de Gemini con el prompt de sistema empático.
6. Gemini genera una respuesta contextualizada y la devuelve.
7. La respuesta aparece en la pantalla de chat.
8. El ciclo se repite hasta que el usuario cierra el chat o vuelve a la pantalla principal.
Postcondiciones La conversación se mantiene en memoria durante la sesión activa. No se persiste en disco ni en servidores externos.
Flujo alternativo FA-01: Si no hay conexión a internet, el sistema muestra un mensaje de error y sugiere usar los recursos disponibles sin conexión.
FA-02: Si la API devuelve un error, se muestra un aviso al usuario y se permite reintentar.
CU-02

Consultar recursos de ayuda

Descripción El estudiante accede a la pantalla de recursos para consultar información de emergencia, guías sobre ciberacoso y técnicas de relajación.
Actores Estudiante
Precondiciones La aplicación está abierta. No requiere conexión a internet (contenido estático).
Flujo normal 1. El estudiante pulsa "Ver recursos" en la pantalla de bienvenida.
2. Se muestra la pantalla de recursos con: números de emergencia (112, ANAR 900 20 20 10), guía sobre ciberacoso y técnicas de relajación.
3. El estudiante lee la información disponible.
4. El estudiante puede volver a la pantalla de bienvenida pulsando el botón de retorno.
Postcondiciones Ninguna. La pantalla es únicamente informativa y estática.
Flujo alternativo No aplica.
CU-03

Visualizar simulación educativa de IA

Descripción El usuario visualiza una simulación educativa en tiempo real donde dos agentes IA interactúan: "Alex" (estudiante víctima de 14 años) y "Tu Amigo" (asistente de apoyo). Las métricas emocionales y el perfil del estudiante se actualizan automáticamente.
Actores Estudiante, Docente / Orientador
Precondiciones La aplicación está abierta y existe conexión a internet para las llamadas a la API de Gemini.
Flujo normal 1. El usuario pulsa "Simulación IA" en la pantalla de bienvenida.
2. Se carga la pantalla con dos paneles: conversación (izquierda) y métricas + perfil (derecha).
3. El usuario pulsa "Iniciar Simulación".
4. El agente víctima ("Alex") genera un mensaje basado en su estado emocional actual.
5. En paralelo, se realiza análisis de perfil del estudiante mediante una segunda llamada a Gemini.
6. Tras 1,5 segundos, el asistente ("Tu Amigo") responde con apoyo empático (máx. 15 palabras).
7. Las barras de métricas emocionales (tristeza, ansiedad, alivio, esperanza) se actualizan en tiempo real.
8. El ciclo se repite cada 2 segundos de forma automática.
9. El usuario puede detener la simulación en cualquier momento.
Postcondiciones La sesión puede ser guardada como archivo JSON descargable (ver CU-04).
Flujo alternativo FA-01: Si la API falla durante el bucle, la simulación se detiene y se muestra el error al usuario.
FA-02: Si el análisis de perfil falla, las métricas anteriores se mantienen.
CU-04

Guardar sesión de simulación

Descripción El usuario descarga un registro completo de la simulación en formato JSON para su posterior análisis o archivo.
Actores Estudiante, Docente / Orientador
Precondiciones Se ha ejecutado al menos un ciclo completo de simulación (hay datos disponibles).
Flujo normal 1. El usuario pulsa "Guardar" durante o después de la simulación.
2. El sistema genera un objeto JSON con: timestamp ISO 8601, perfil del estudiante, métricas emocionales y lista completa de mensajes.
3. El navegador/sistema operativo descarga el archivo automáticamente.
Postcondiciones El archivo JSON queda guardado localmente en el sistema del usuario.
Flujo alternativo No aplica.

Áreas de Gestión del Proyecto

Gestión del Alcance

El proyecto se delimitó explícitamente al desarrollo del MVP de escritorio. Las funcionalidades de backend (autenticación, base de datos, generación de informes PDF, notificación a centros educativos) quedaron fuera del alcance actual y se documentan como hoja de ruta para versiones futuras.

Gestión de Riesgos

RiesgoProbabilidadImpactoMitigación
Dependencia de API externa (Gemini) Media Alto Mensajes de error claros al usuario; posibilidad de cambiar modelo en configuración.
Exposición de datos sensibles de menores Baja Muy alto Toda la conversación permanece en memoria local. La API Key se gestiona exclusivamente en el proceso principal, nunca en el renderer.
Impacto emocional negativo en el usuario Baja Alto Prompt de sistema diseñado para derivar a profesionales en situaciones de riesgo alto. Recursos de emergencia siempre accesibles.
Incompatibilidad entre versiones de Electron Baja Medio Versiones fijadas en package.json. Tests en Windows, macOS y Linux.

Gestión de la Calidad

Para garantizar la calidad del código se aplicaron las siguientes medidas:

TypeScript estricto
Tipado completo de todas las interfaces y componentes. Configuración strict: true en tsconfig.json.
ESLint 9
Reglas de linting configuradas para React + TypeScript. Integrado en el flujo de desarrollo.
ErrorBoundary React
Componente ErrorBoundary.tsx que captura errores en el árbol de componentes y muestra una interfaz de fallback amigable.
Seguridad IPC
Superficie de API mínima expuesta mediante contextBridge. El renderer nunca accede directamente a módulos de Node.js.
Pruebas de usabilidad
Validación del flujo de usuario con el perfil objetivo (estudiantes de 12–18 años).

Fases de Implementación

1. Configuración del entorno Electron + React

Se configuró el proyecto con Vite como bundler del proceso renderer y TypeScript para ambos procesos (renderer y main). Se estableció la configuración de concurrently para arrancar Vite y Electron en paralelo durante el desarrollo.

2. Arquitectura IPC segura

Se implementó el puente seguro preload.ts utilizando contextBridge.exposeInMainWorld() para exponer únicamente la función chat() al renderer. Esta superficie mínima de API garantiza que el proceso renderer no tenga acceso a módulos de Node.js ni a la API Key de Gemini.

3. Integración con Google Gemini

El proceso principal gestiona todas las llamadas a @google/generative-ai. Se implementó un algoritmo de normalización del historial de mensajes que garantiza el formato requerido por Gemini: alternancia estricta de roles user / model, sin mensajes consecutivos del mismo rol, y siempre comenzando por el rol user.

4. Pantallas React con hooks

Los cuatro componentes principales (WelcomeScreen, ChatScreen, ResourcesScreen, SimulationScreen) se implementaron como componentes funcionales React con gestión de estado mediante useState, efectos secundarios con useEffect y referencias de DOM con useRef (para scroll automático al último mensaje).

5. Simulación multi-agente

La SimulationScreen implementa un bucle asíncrono con dos agentes LLM distintos. Las métricas emocionales se inyectan dinámicamente en el prompt del agente víctima en cada ciclo. El análisis de perfil del estudiante se ejecuta en paralelo (sin bloquear el hilo principal de la UI) mediante Promise.all(). Las respuestas del asistente se dividen en múltiples mensajes usando el separador |, simulando un delay de escritura proporcional a la longitud de la respuesta.

💡

Decisión de diseño clave: Se optó por no persistir las conversaciones en disco ni en servidores para el MVP. Esto elimina cualquier riesgo de exposición de datos sensibles de menores de edad y simplifica la arquitectura, alineándose con el principio de privacy by design del RGPD.

Innovación

Tu Amigo aporta innovación en varios niveles técnicos y conceptuales respecto a las soluciones existentes en el mercado educativo:

#Aspecto innovadorDescripción
1 LLM para apoyo emocional contextualizado A diferencia de chatbots basados en reglas o árboles de decisión, la app usa Gemini 2.0 Flash con un prompt de sistema diseñado específicamente para contención emocional en situaciones de acoso. La IA adapta su tono y contenido a cada conversación de forma dinámica.
2 Simulación multi-agente educativa en tiempo real La SimulationScreen implementa dos agentes LLM con personalidades y objetivos distintos que interactúan de forma autónoma. Esto es poco común en aplicaciones educativas para este rango de edad y tiene un alto potencial como herramienta de formación para docentes y orientadores.
3 Métricas emocionales dinámicas generadas por IA El sistema extrae métricas psicológicas (tristeza, ansiedad, alivio, esperanza) directamente de las respuestas del agente IA mediante análisis estructurado (JSON), sin necesidad de modelos de clasificación emocional separados. Se actualiza en cada ciclo de simulación.
4 Privacidad por diseño (Privacy by Design) Toda la conversación se mantiene en memoria local del proceso Electron. La API Key nunca se expone al renderer. Cero datos personales enviados a servidores propios. Alineación completa con los principios de minimización de datos del RGPD europeo.
5 Arquitectura Electron multiplataforma Un único codebase TypeScript/React genera instaladores nativos para Windows (.exe), macOS (.dmg) y Linux (.AppImage), facilitando la distribución en centros educativos con cualquier sistema operativo, sin dependencia de navegador ni conexión permanente.
V

Resultados y Conclusiones

Resultados Obtenidos

🖥️ App multiplataforma funcional 💬 Chat empático con IA operativo 📊 Métricas emocionales en tiempo real 🎭 Simulación multi-agente completa

El proyecto Tu Amigo demuestra que es técnicamente viable construir una herramienta de apoyo emocional para estudiantes víctimas de acoso escolar usando tecnologías modernas de inteligencia artificial y desarrollo de escritorio, sin comprometer la privacidad de los usuarios.

Conclusiones

Los LLMs ofrecen capacidad empática suficiente para una primera capa de apoyo
Gemini 2.0 Flash genera respuestas contextualmente apropiadas y emocionalmente inteligentes, aunque no sustituyen a un profesional de la psicología. La app lo comunica explícitamente.
Electron simplifica la distribución en entornos educativos
Un instalador nativo por plataforma elimina las barreras de adopción en centros con infraestructura heterogénea, sin depender de servidores propios para el MVP.
La privacidad debe ser un pilar desde el diseño inicial
Cuando los usuarios son menores de edad en situaciones de vulnerabilidad, la arquitectura técnica debe garantizar la confidencialidad por diseño, no como capa añadida a posteriori.
La simulación multi-agente tiene alto potencial formativo
Los docentes y orientadores pueden usar la pantalla de simulación para comprender cómo responde el sistema antes de recomendarlo a estudiantes, aumentando la confianza en la herramienta.
